# Transformation Lifecycle

## 1. Input Acquisition
Raw text, structured data, or mixed formats.

## 2. Context Normalisation
Tokenisation, segmentation, metadata extraction.

## 3. Parsing
Converts input into an intermediate semantic structure.

## 4. Resolution
Applies domain rules, resolves ambiguities, enforces constraints.

## 5. Mapping
Converts resolved structures into target schemas.

## 6. Validation
Ensures completeness, correctness, and rule compliance.

## 7. Serialisation
Outputs JSON, YAML, XML, or domain‑specific formats.

## 8. Trace Generation
Full audit trail of decisions, rules applied, and transformations.
