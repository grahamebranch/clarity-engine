# 6. Canonical vs Derived Files

### **Canonical (stored in repo):**
- engine phases  
- operational parameters  
- business context  
- business values  
- recovery protocol  
- snapshot manifest  
- completeness analysis  

### **Derived (regenerated, never stored):**
- roadmap  
- forecast  
- pricing model  
- adoption curves  
- risk models  
- scenario outputs  

Derived views must always be regenerated from the four layers.
