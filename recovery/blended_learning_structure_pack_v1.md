# Blended Learning Structure Pack (v1) — Clarity Engine

## 1. Purpose
The Blended Learning Structure Pack defines the structures, rules, and templates required to generate **hybrid learning experiences** that combine:

- synchronous lessons  
- asynchronous tasks  
- self‑study components  
- trainer‑supported activities  

This is a Phase‑4 extension and does not modify:
- system invariants
- lesson structure
- CEFR rules
- governance
- edition isolation

---

## 2. Structure Identity

### 2.1 Name
**Blended Learning Structure Pack**

### 2.2 Scope
Supports:
- hybrid corporate programmes  
- school/university blended courses  
- flipped‑classroom models  
- asynchronous + synchronous cycles  
- self‑study reinforcement  

### 2.3 Output Types
- blended lesson plan  
- asynchronous task set  
- synchronous session plan  
- integration notes  
- trainer guidance (if edition allows)  

---

## 3. Blended Structure Model (Authoritative)

All blended learning outputs must follow this structure:

1. **Overview**  
2. **Learning Outcomes**  
3. **Asynchronous Component**  
4. **Synchronous Component**  
5. **Integration Notes**  
6. **Assessment Points**  
7. **Edition‑Specific Notes**  

---

---

## 6. Integration Rules

### 6.1 Asynchronous → Synchronous
Synchronous sessions must:
- build on asynchronous tasks  
- recycle vocabulary  
- reinforce functional language  
- address learner difficulties  

### 6.2 Synchronous → Asynchronous
Asynchronous tasks must:
- extend lesson outcomes  
- provide additional practice  
- prepare for next session  

### 6.3 Cohesion Requirements
Blended programmes must:
- maintain consistent CEFR alignment  
- maintain consistent learning outcomes  
- avoid duplication  
- avoid cognitive overload  

---

## 7. Assessment Points

Assessment must include:
- asynchronous micro‑checks  
- synchronous performance tasks  
- periodic consolidation tasks  
- optional self‑assessment

## 9. Validation Rules

Blended learning outputs must:
- follow CEFR  
- follow edition rules  
- follow domain isolation  
- avoid bias  
- avoid cultural assumptions  

Forbidden:
- real company data  
- personal data retention  
- legal/financial advice
