# Accessibility Edition (v1) — Clarity Engine

## 1. Purpose
The Accessibility Edition adapts lesson content to support learners with:

- cognitive processing differences  
- attention‑related challenges  
- reading difficulties  
- visual impairments (mild–moderate)  
- language‑processing difficulties  
- neurodiverse learning profiles  

This edition ensures **maximum clarity, reduced cognitive load, and inclusive design** while preserving:

- lesson structure  
- CEFR alignment  
- semantic meaning  
- domain isolation  
- edition isolation  

This is a Phase‑4 extension and does not modify system‑layer behaviour.

---

## 2. Edition Identity

### 2.1 Name
**Accessibility Edition**

### 2.2 Audience
- neurodiverse learners  
- learners with mild–moderate dyslexia  
- learners with attention challenges  
- learners needing simplified visual structure  
- learners needing reduced cognitive load  

### 2.3 Purpose
To provide:
- simplified formatting  
- reduced density  
- clearer sequencing  
- explicit instructions  
- predictable structure  
- enhanced readability  

---

## 3. Edition Invariants

Accessibility Edition must:
- preserve lesson structure  
- preserve CEFR alignment  
- preserve semantic meaning  
- remain edition‑isolated  
- avoid infantilisation  
- avoid oversimplification of content meaning  

Accessibility Edition may **simplify presentation**, but never **simplify the underlying learning target**.

---

## 4. Tone & Voice

### 4.1 Tone
- calm  
- clear  
- supportive  
- neutral  

### 4.2 Voice
- direct  
- step‑by‑step  
- non‑overwhelming  
- confidence‑building  

---

## 5. Content Expansions

Accessibility Edition must include:

### 5.1 Explicit Instructions
- numbered steps  
- clear task boundaries  
- explicit expectations  

### 5.2 Reduced Cognitive Load
- shorter sentences  
- simplified formatting  
- fewer items per block  
- increased white‑space  

### 5.3 Visual Clarity
- clear section headers  
- consistent spacing  
- minimal inline formatting  

### 5.4 Processing Support
- pre‑teaching key vocabulary  
- chunking tasks  
- optional scaffolding prompts  

---

## 6. Content Suppressions

Accessibility Edition must NOT include:
- dense paragraphs  
- complex formatting  
- idiomatic overload  
- rapid task switching  
- unnecessary decoration  

---

## 7. Formatting Rules

### 7.1 Accessibility Blocks
Accessibility‑specific content must be clearly marked:

**ACCESS TIP:**  
**STEP‑BY‑STEP:**  
**READING SUPPORT:**  

### 7.2 Structure
- short blocks  
- clear spacing  
- minimal nested lists  

### 7.3 Placement
Accessibility blocks appear:
- before tasks  
- before examples  
- at the start of sections  

---

## 8. Edition Interaction

### 8.1 With Trainer Edition
Trainer Edition may include accessibility coaching notes, but Accessibility Edition remains learner‑facing.

### 8.2 With Client Edition
Accessibility Edition is a specialised variant of Client Edition with:
- more structure  
- more clarity  
- more processing support  

### 8.3 With Youth Edition
Accessibility Edition may borrow Youth Edition scaffolding **only when appropriate for cognitive load**, not age.

### 8.4 With Cultural Edition
Cultural notes must be simplified and optional.

---

## 9. Validation Rules

Accessibility Edition must pass:
- OSF validation  
- DIS validation  
- governance validation  
- edition isolation validation  
- CEFR alignment validation  

Forbidden:
- medical claims  
- diagnostic language  
- therapeutic framing  
- assumptions about disability  

---

## 10. Summary
The Accessibility Edition provides a **clear, structured, cognitively supportive** version of each lesson.  
It is fully compliant with:

- system invariants  
- governance  
- meta rules  
- lesson rules  
- edition rules  

This is the sixth major extension in Phase‑4.
