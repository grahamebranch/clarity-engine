# Travel English Domain Pack (v1) — Clarity Engine

## 1. Purpose
The Travel English Domain Pack defines the rules, templates, flows, and vocabulary required to generate CEFR‑aligned travel‑communication lessons.

This is a Phase‑4 extension and does not modify:
- system invariants
- lesson structure
- CEFR rules
- governance
- edition isolation

---

## 2. Domain Identity

### 2.1 Name
**Travel English Domain**

### 2.2 Scope
Covers communication in:
- airports
- hotels
- restaurants
- transport
- emergencies
- sightseeing
- shopping
- directions

### 2.3 CEFR Range
A1 → B2  
(C‑levels are excluded because travel communication rarely requires advanced academic or professional structures.)

---

## 3. Domain Rules

### 3.1 Domain Invariants
Travel English lessons must:
- follow the global lesson structure
- include functional survival language
- include situational vocabulary
- include short, practical tasks
- prioritise clarity over complexity

### 3.2 Domain Restrictions
Travel English lessons must NOT:
- include legal/medical advice
- include real flight or booking data
- include political or cultural stereotypes
- include advanced business language

### 3.3 Domain Alignment
All tasks must be:
- practical
- scenario‑based
- short‑form
- CEFR‑appropriate

---

## 4. Domain Templates

### 4.1 Airport Template
Includes:
- check‑in language
- security questions
- boarding announcements
- problem‑solving (lost luggage)

### 4.2 Hotel Template
Includes:
- booking
- check‑in/out
- complaints
- requests

### 4.3 Restaurant Template
Includes:
- ordering
- dietary needs
- complaints
- paying

### 4.4 Transport Template
Includes:
- buying tickets
- asking for directions
- delays
- route clarification

---

## 5. Domain Flows

### 5.1 Airport Flow
1. Warm‑up scenario  
2. Key phrases  
3. Controlled dialogue  
4. Problem‑solving task  
5. Roleplay  

### 5.2 Hotel Flow
1. Identify purpose  
2. Learn key structures  
3. Practice requests  
4. Handle problems  
5. Final roleplay  

### 5.3 Restaurant Flow
1. Menu vocabulary  
2. Ordering language  
3. Clarification  
4. Complaint handling  
5. Payment  

---

## 6. Domain Vocabulary Sets

### 6.1 Core Travel Vocabulary (A1–A2)
- ticket  
- passport  
- reservation  
- luggage  
- platform  
- menu  
- bill  

### 6.2 Functional Phrases (A2–B1)
- “Where is the…?”  
- “I’d like to…”  
- “Can you help me with…?”  
- “How much is…?”  

### 6.3 Problem‑Solving Phrases (B1–B2)
- “There seems to be a mistake…”  
- “I think there’s a misunderstanding…”  
- “Could you check again, please?”  

---

## 7. Functional Language Sets

### 7.1 Asking for Help
- “Excuse me, could you help me…?”  
- “I’m looking for…”  

### 7.2 Clarifying
- “Do you mean…?”  
- “Could you repeat that?”  

### 7.3 Emergencies
- “I need help.”  
- “I’ve lost my…”  
- “Call the police/ambulance.”  

---

## 8. Cultural Notes
Travel English lessons must include:
- politeness norms  
- tipping norms (neutral, non‑prescriptive)  
- direct vs indirect requests  
- international English variations  

---

## 9. Validation Rules

Travel English outputs must:
- follow CEFR  
- follow edition rules  
- avoid stereotypes  
- avoid cultural assumptions  
- avoid real booking data

Forbidden:
- legal advice  
- medical advice  
- personal data retention  

---

## 10. Summary
The Travel English Domain Pack provides the structures, templates, flows, vocabulary, and functional language required to generate practical travel‑communication lessons.

It is fully compliant with:
- system invariants
- governance
- meta rules
- lesson rules
- edition rules
