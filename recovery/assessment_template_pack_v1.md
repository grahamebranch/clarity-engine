# Assessment Template Pack (v1) — Clarity Engine

## 1. Purpose
The Assessment Template Pack defines the **structures, rules, and templates** used to generate CEFR‑aligned assessments inside the Clarity Engine.

This is a Phase‑4 extension and does not modify:
- system invariants  
- lesson structure  
- edition rules  
- governance  
- CEFR definitions  

---

## 2. Assessment Identity

### 2.1 Name
**Assessment Template Pack**

### 2.2 Scope
Supports:
- placement tests  
- end‑of‑unit assessments  
- progress checks  
- skill‑specific assessments  
- corporate performance assessments  

### 2.3 CEFR Range
A1 → C2  
(Templates adapt automatically by level.)

---

## 3. Assessment Structure (Authoritative)

All assessments must follow this structure:

1. **Purpose**  
2. **Instructions**  
3. **Task Set**  
4. **Scoring Guide**  
5. **Performance Indicators**  
6. **CEFR Mapping**  
7. **Trainer Notes (if edition allows)**  

---

## 4. Assessment Templates

### 4.1 Speaking Assessment Template

- warm‑up question

- problem‑solving task  
- extended response  
- follow‑up questions

- fluency

- interaction  
- coherence

## 8. Edition Interaction

### 8.1 Trainer Edition
Includes:
- scoring guidance  
- error patterns  
- feedback strategies  

### 8.2 Client Edition
Includes:
- simplified instructions  
- clear examples  
- no scoring rubric  

### 8.3 Corporate Trainer Edition
Includes:
- performance indicators  
- workplace alignment  
- coaching strategies  

### 8.4 Cultural Edition
Includes:
- cultural tone notes  
- register differences  

---

## 9. Validation Rules

Assessments must:
- follow CEFR  
- follow edition rules  
- avoid bias  
- avoid cultural assumptions  
- avoid confidential corporate content  

Forbidden:
- real company data  
- legal/financial advice  
- personal data retention  

---

## 10. Summary
The Assessment Template Pack provides the **structures, templates, scoring guides, and CEFR‑aligned rules** required to generate assessments across all domains and editions.

It is fully compliant with:
- system invariants  
- governance  
- meta rules  
- lesson rules  
- edition rules
